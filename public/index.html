Documentation

You have to make sure that the routes you want to test exist in this MockServer for this to work. Each route and data point is setup manually in here. So if you are testing a brand new endpoint, it will not automagically appear in this server.

Endpoint Details

  • Orders
    • GET /v1/orders -Returns a list of orders from db.json file.
    • GET /v1/orders/1 -Returns the order based on the order number from db.json.
    • PUT PUT /v1/orders/1 -Processes request and creates an order in the db.json. The record that is created is formated to return the expected results that includes shipping number and tracking id's. The results are returned in the request response. The process does validate shipTo, billTo, quantity, and if the prodct exist in the valid list of product id's and will return a 404 if one of the valiations fails.
  • Returns
    • POST /v1/returns -Adds return json into db.json file.
    • GET /v1/returns?returnedAfter='2025-01-27' -Gets records from db.json file where the returndAfter date is = to the date provided in the query.
  • Shippmints
      • Tracking Numbers: If you want to see an order that has multiple tracking numbers, do the following. This can be done by setting the quantity on the sales order line item to a given amonut.
        -1 Tracking Number: set the item quantity to less tha equal to 10.
        -2 Tracking Numbers: set the quantity to equals 20
        -3 Tracking Numbers: set the quantity to equals 30

    Supported End to End Processes

    Supported End to End Processes
    Process Endpoint(s) Description
    export-orders PUT /orders The mock server returns a 201 status code for regular orders to indicate that the order has been sent to TR and that the "sentToWarehouse" falg is set to true.
    export-transfer-orders PUT /orders The mock server returns a 201 status code for transfer orders to indicate that the order has been sent and that the "sentToWarehouse" falg is set to true.
    export-subscriptions PUT /orders The mock server returns a 201 status code for subscription orders to indicate that the order has been sent and that the "sentToWarehouse" falg is set to true.
    shipping:order-confirmation GET /orders The mock server returns a 200 status code for regulare and subscription orders has been return. The shpping and tracking number in contained in the resonse.
